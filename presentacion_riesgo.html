Diseño de la API REST

La API se construyó con FastAPI usando una arquitectura modular. La app crea el `FastAPI` principal, configura CORS y monta routers especializados para cada dominio.

  • Inyección de dependencias: `Depends(get_settings)` y `Depends(get_data_service)` separan configuración, servicios y controladores.
  • Routers por dominio: `router_activos`, `router_precios`, `router_rendimientos`, `router_var`, `router_capm`, `router_macro`.
  • Escalabilidad: los endpoints delegan en servicios como `DataService`, `RiskCalculator` y `MacroService`.
main.py - Estructura real 
def create_app() -> FastAPI:
    settings = get_settings()

    app = FastAPI(
        title=settings.app_name,
        version=settings.app_version,
        description="API de análisis de riesgo financiero — Proyecto Integrador USTA.",
        docs_url="/docs",
        redoc_url="/redoc",
        debug=settings.debug,
    )

    app.add_middleware(
        CORSMiddleware,
        allow_origins=settings.cors_origins,
        allow_credentials=True,
        allow_methods=["*"],
        allow_headers=["*"],
    )

    app.include_router(router_activos)
    app.include_router(router_precios)
    app.include_router(router_rendimientos)
    app.include_router(router_var)
    app.include_router(router_capm)
    app.include_router(router_macro)

    return app
backend/app/models.py

Pydantic: Validando la Teoría

En Teoría del Riesgo, si los pesos de un portafolio no suman 1.0, el cálculo de VaR es inválido. Usé validadores de Pydantic para garantizar consistencia antes de ejecutar el servicio.

  • BaseModel: define `VaRRequest`, `VaRResultado` y `VaRResponse`.
  • @field_validator: normaliza tickers y valida que todos los pesos sean mayores o iguales a cero.
  • @model_validator(mode="after"): asegura que `len(tickers) == len(pesos)` y que la suma de pesos sea 1.0 con tolerancia ±0.01.
models.py - Validación de VaR 
class VaRRequest(BaseModel):
    tickers: list[str] = Field(...)
    pesos: list[float] = Field(...)
    nivel_confianza: float = Field(default=0.95, ge=0.90, le=0.999)

    @field_validator("tickers")
    def tickers_en_mayusculas(cls, v: list[str]) -> list[str]:
        return [t.upper().strip() for t in v]

    @field_validator("pesos")
    def pesos_positivos(cls, v: list[float]) -> list[float]:
        if any(p < 0 for p in v):
            raise ValueError("Todos los pesos deben ser mayores o iguales a 0.")
        return v

    @model_validator(mode="after")
    def validar_consistencia_portafolio(self) -> "VaRRequest":
        if len(self.tickers) != len(self.pesos):
            raise ValueError("La cantidad de tickers debe coincidir con la cantidad de pesos.")

        suma = sum(self.pesos)
        if abs(suma - 1.0) > 0.01:
            raise ValueError(f"Los pesos deben sumar 1.0. Suma actual: {suma:.4f}.")
        return self

Ingesta y Limpieza de Series

El servicio `DataService` descarga precios ajustados de Yahoo Finance, normaliza columnas y construye rendimientos simples y logarítmicos listos para el cálculo de riesgo.

  • Auto-ajustado: descarga OHLCV con `auto_adjust=True` para incluir splits y dividendos.
  • Retornos log: `np.log(cierre / cierre.shift(1))` para cumplir con las propiedades de aditividad temporal.
  • Limpieza: `ffill()` y `dropna()` eliminan huecos por fines de semana y feriados.
services.py - Cálculo de rendimientos 
def get_rendimientos(
    self,
    ticker: str,
    fecha_inicio: Optional[date] = None,
    fecha_fin: Optional[date] = None,
) -> pd.DataFrame:
    df_precios = self.get_precios(ticker, fecha_inicio, fecha_fin)
    cierre = df_precios["cierre"]

    df_rend = pd.DataFrame(index=cierre.index)
    df_rend["rendimiento_simple"] = cierre.pct_change()
    df_rend["rendimiento_log"] = np.log(cierre / cierre.shift(1))

    df_rend = df_rend.dropna()
    return df_rend

Estadística Descriptiva y Pruebas

Antes de medir riesgo se evalúa si los rendimientos cumplen la hipótesis de normalidad. El backend usa `Jarque-Bera` y `Shapiro-Wilk` para detectar cola pesada y asimetría.

  • Momentos clave: media, desviación estándar, asimetría y curtosis.
  • Prueba JB: compara asimetría y curtosis con la distribución normal.
  • Shapiro-Wilk: refuerza la conclusión de normalidad en la muestra.
services.py - Pruebas de normalidad 
def calcular_estadisticas(self, rendimientos: pd.Series) -> dict:
    jb_stat, jb_pvalue = stats.jarque_bera(rendimientos.dropna())

    muestra_sw = rendimientos.dropna()
    if len(muestra_sw) > 5000:
        muestra_sw = muestra_sw.sample(5000, random_state=42)
    sw_stat, sw_pvalue = stats.shapiro(muestra_sw)

    return {
        "media": float(rendimientos.mean()),
        "desviacion_std": float(rendimientos.std()),
        "asimetria": float(rendimientos.skew()),
        "curtosis": float(rendimientos.kurtosis()),
        "jarque_bera_pvalue": float(jb_pvalue),
        "shapiro_wilk_pvalue": float(sw_pvalue),
        "es_normal": bool(jb_pvalue > 0.05 and sw_pvalue > 0.05),
    }
backend/app/services.py

Motores de Cálculo de Riesgo

La clase `RiskCalculator` ejecuta los tres métodos de VaR y recomienda el más robusto según las propiedades reales de los rendimientos.

  • VaR paramétrico: asume normalidad e implementa la raíz del tiempo.
  • VaR histórico: usa el percentil empírico de la distribución real.
  • VaR Monte Carlo: simula n_simulaciones=10_000 escenarios para capturar extremos.
services.py - Selección de método 
def calcular_var_completo(
    self,
    rendimientos_portafolio: pd.Series,
    nivel_confianza: float = 0.95,
    horizonte_dias: int = 1,
    valor_portafolio: float = 10_000.0,
) -> dict:
    resultados = [
        self.var_parametrico(rendimientos_portafolio, nivel_confianza, horizonte_dias, valor_portafolio),
        self.var_historico(rendimientos_portafolio, nivel_confianza, horizonte_dias, valor_portafolio),
        self.var_montecarlo(rendimientos_portafolio, nivel_confianza, horizonte_dias, valor_portafolio),
    ]

    _, jb_pvalue = stats.jarque_bera(rendimientos_portafolio.dropna())
    if jb_pvalue > 0.05:
        recomendado = "parametrico"
    else:
        recomendado = "historico"

    return {
        "resultados": resultados,
        "mejor_metodo_recomendado": recomendado,
    }

GARCH, CAPM y FRED

El backend no solo calcula VaR; también ajusta modelos de volatilidad condicional y consume datos macro desde FRED para obtener tasas libres de riesgo dinámicas.

  • GARCHService: ajusta `ARCH(1)`, `GARCH(1,1)` y `EGARCH(1,1)` y compara AIC/BIC.
  • MacroService: obtiene tasa libre de riesgo desde `DGS3MO` y mide inflación con `CPIAUCSL`.
  • CAPM/Markowitz: los resultados se integran con el dashboard para explicar Beta, Alpha y portafolios eficientes.
services.py - FRED y volatilidad 
async def get_tasa_libre_riesgo(self) -> float:
    observaciones = await self.fred_service.get_series("DGS3MO", limit=1)
    if not observaciones:
        logger.warning("Sin datos de FRED. Usando tasa por defecto 4.5%.")
        return 4.5

    try:
        valor = float(observaciones[0]["value"])
        return valor
    except (ValueError, KeyError, IndexError):
        logger.warning("Error parseando tasa de FRED. Usando 4.5%.")
        return 4.5

El Consumidor: Streamlit

La UI usa Streamlit como cliente HTTP. El frontend crea el payload, envía el request, y presenta los resultados de VaR/CVaR y los módulos de riesgos.

  • API helpers: `api_post()` encapsula `requests.post(..., json=body)` con manejo de errores.
  • Validación visual: el dashboard advierte si los pesos no suman 1.0 antes de enviar la petición.
  • Resultados: muestra `data_var["mejor_metodo_recomendado"]` y tabla con VaR/CVaR de cada método.
