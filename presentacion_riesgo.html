Diseño de la API REST

El proyecto trasciende un simple script lineal. Implementé una arquitectura modular basada en el patrón Modelo-Vista-Controlador (MVC) adaptado a microservicios.

  • Inyección de Dependencias: Uso de Depends() para aislar la lógica de configuración y clientes HTTP.
  • APIRouter: División del dominio en submódulos lógicos (/riesgo, /mercado), evitando archivos monolíticos.
  • Asincronismo: Endpoints definidos con async def para no bloquear el Event Loop al hacer peticiones a Yahoo Finance.
main.py - Estructura de Endpoints 
from fastapi import FastAPI, APIRouter, Depends
from app.config import Settings, get_settings
from app.api.endpoints import riesgo, mercado

app = FastAPI(title="RiskLab USTA API")
api_router = APIRouter()

# Modularidad: Enganchando sub-rutas
api_router.include_router(riesgo.router, prefix="/riesgo")
api_router.include_router(mercado.router, prefix="/mercado")

# Montando la API completa con un prefijo de versión
app.include_router(api_router, prefix="/api/v1")

@app.get("/health")
async def health_check(settings: Settings = Depends(get_settings)):
    return {"status": "ok", "version": settings.APP_VERSION}
backend/app/models.py

Pydantic: Validando la Teoría

En Teoría del Riesgo, si los pesos de un portafolio no suman 100%, la optimización de Markowitz colapsa. Usé Decoradores de Pydantic para blindar el modelo matemático.

  • BaseModel: Tipado fuerte para Requests y Responses.
  • @model_validator(mode='after'): Este decorador se ejecuta automáticamente después de instanciar la clase.
  • Protección Lógica: Rechaza la petición (Error 422) antes de que toque los motores de cálculo, ahorrando procesamiento.
models.py - Decoradores 
from pydantic import BaseModel, model_validator
from typing import List

class PortafolioRequest(BaseModel):
    activos: List[str]
    pesos: List[float]

    @model_validator(mode='after')
    def validar_pesos_markowitz(self):
        # Lógica Core: Validar restricción presupuestaria
        suma_pesos = sum(self.pesos)
        if abs(suma_pesos - 1.0) > 1e-4:
            raise ValueError("Error: Los pesos deben sumar exactamente 1.0 (100%)")
        return self
backend/app/services.py

Ingesta y Limpieza de Series

El Módulo 1 requiere obtener datos de mínimo 5 activos. Desarrollé el DataService utilizando Pandas para transformar los datos crudos de la API en información modelable.

  • Vectorización: Uso de operaciones vectorizadas (np.log) en lugar de bucles for para máxima eficiencia.
  • Rendimientos Continuos: Cálculo matemático del diferencial logarítmico.
  • Manejo de NaNs: Limpieza automatizada de días no bursátiles (feriados) usando .dropna().
services.py - DataService 
import yfinance as yf
import pandas as pd
import numpy as np

class DataService:
    async def get_market_data(self, tickers: List[str], period: str = "3y"):
        # Descarga masiva de múltiples activos
        data = yf.download(tickers, period=period)['Close']
        
        # Cálculo Vectorizado de Retornos Logarítmicos: ln(Pt / Pt-1)
        returns = np.log(data / data.shift(1))
        
        # Limpieza de valores nulos del primer día
        return returns.dropna()
Requisito 3: Propiedades Empíricas

Estadística Descriptiva y Pruebas

Para evaluar el riesgo, primero hay que entender la distribución. Usando scipy.stats, el backend ejecuta pruebas estadísticas formales para detectar colas pesadas y asimetría.

  • Momentos: Media, Varianza, Asimetría (Skew) y Curtosis (Kurtosis).
  • Jarque-Bera (JB): Prueba de bondad de ajuste que evalúa si la asimetría y la curtosis corresponden a una Normal.
  • Shapiro-Wilk: Validación cruzada de la normalidad para muestras específicas.
services.py - Pruebas Estadísticas 
from scipy.stats import skew, kurtosis, jarque_bera, shapiro

def calcular_estadisticas(self, serie_retornos):
    # Análisis de forma de la distribución
    asimetria = skew(serie_retornos)
    curt_exceso = kurtosis(serie_retornos) # Base 3
    
    # Prueba Jarque-Bera
    jb_stat, jb_p_value = jarque_bera(serie_retornos)
    
    # Interpretación algorítmica para el Frontend
    es_normal_jb = True if jb_p_value > 0.05 else False
    
    return {
        "asimetria": asimetria,
        "curtosis": curt_exceso,
        "p_value_jb": jb_p_value,
        "distribucion_normal": es_normal_jb
    }
Requisito 5: VaR y CVaR

Motores de Cálculo de Riesgo

El corazón matemático del proyecto. La clase RiskCalculator consolida 3 metodologías distintas para estimar la Máxima Pérdida Esperada (VaR) y el Expected Shortfall.

  • VaR Paramétrico: Usa norm.ppf() (función de punto porcentual) asumiendo campana de Gauss.
  • VaR Histórico: Uso empírico de np.percentile() sobre la historia real.
  • CVaR (Expected Shortfall): Promedio condicional de las pérdidas que superan el umbral del VaR.
services.py - RiskCalculator 
from scipy.stats import norm

class RiskCalculator:
    def var_parametrico(self, retornos, alpha=0.05):
        # Inversa de la función de distribución acumulada (Z-score)
        z_score = norm.ppf(alpha)
        return np.mean(retornos) + (z_score * np.std(retornos))
        
    def var_historico(self, retornos, alpha=0.05):
        # Búsqueda del cuantil empírico real
        return np.percentile(retornos, alpha * 100)
        
    def cvar_historico(self, retornos, alpha=0.05):
        # Filtro booleano: Promedio de la cola izquierda extrema
        var = self.var_historico(retornos, alpha)
        return retornos[retornos <= var].mean()
Requisito 6, 7 y Bono

CAPM, Markowitz y Macroeconomía

No usamos datos estáticos ("quemados"). La API se conecta al Federal Reserve Economic Data (FRED) para obtener la Tasa Libre de Riesgo (US Treasuries de 10 años) dinámica.

  • Beta (β): Covarianza del activo vs. mercado / Varianza del mercado.
  • Markowitz: Construcción de la matriz de varianzas-covarianzas para simular portafolios eficientes.
  • Tracking Error: Comparativa algorítmica del Ratio de Sharpe del portafolio vs Benchmark (^GSPC).
services.py - Macro Integrations 
import requests

class MacroService:
    def get_risk_free_rate(self):
        # Consumo de API Oficial (Bono del Proyecto)
        # DGS10: 10-Year Treasury Constant Maturity Rate
        url = f"https://api.stlouisfed.org/fred/series/observations?series_id=DGS10..."
        resp = requests.get(url).json()
        
        # Limpieza y conversión a formato decimal
        tasa_porcentual = float(resp['observations'][-1]['value'])
        return tasa_porcentual / 100.0
